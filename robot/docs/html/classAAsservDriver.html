Description détaillée

Interface abstraite du driver d'asservissement.

Regroupe les commandes de mouvement (goto, rotation, ligne), le controle direct des moteurs/encodeurs, et la gestion de l'odometrie. Chaque plateforme (ARM, SIMU, EsialR) fournit sa propre implementation.

Note
Cette interface contient 47 methodes et sera a terme decoupee en AAsserv (haut niveau) + AMotorDriver (bas niveau). Voir ARCHITECTURE.md pour le plan de refactoring.
